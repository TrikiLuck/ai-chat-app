# Исправления и обновления

## 2026-04-06 - Блокировка загрузки изображений

### Проблема
Модель не поддерживает обработку изображений, но в коде была возможность загружать любые файлы, включая изображения.

### Решение
✅ Добавлена валидация при загрузке файлов - блокируются все файлы с MIME-типом `image/*`
✅ Добавлена проверка в контроллере сообщений - если к сообщению прикреплены изображения, запрос отклоняется
✅ Обновлена документация - указано, что поддерживаются только текстовые файлы
✅ Добавлены понятные сообщения об ошибках для пользователей

### Изменённые файлы
- `backend/src/controllers/fileController.ts` - валидация при загрузке
- `backend/src/controllers/messageController.ts` - проверка перед отправкой
- `README.md` - обновлена документация

## Как это работает

**При загрузке файла:**
1. Пользователь пытается загрузить файл
2. Backend проверяет MIME-тип
3. Если это изображение - файл удаляется и возвращается ошибка
4. Если текстовый файл - загрузка проходит успешно

**При отправке сообщения с файлами:**
1. Проверяются все прикреплённые файлы
2. Если найдено изображение - сообщение не отправляется
3. Возвращается понятная ошибка пользователю

### Поддерживаемые типы файлов
✅ Текстовые файлы (.txt, .md, .json, .csv и т.д.)
✅ Документы (.pdf, .doc, .docx)
✅ Код (.js, .py, .java и т.д.)
❌ Изображения (.png, .jpg, .gif и т.д.)
❌ Видео
❌ Аудио
